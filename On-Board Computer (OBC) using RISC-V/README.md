# On-Board Computer (OBC) using RISC-V

## Overview
The legacy on-board computers were based on **ADSP processors** from Analog Devices, while the new series of systems used **Microchip PIC32**. Although these platforms were reliable and well-established for prototypes, they posed challenges in terms of availability and lifetime.

RISC-V, on the other hand, offered flexibility, as it can be customized per requirements, implemented on any FPGA, and potentially lead to developing an in-house processor. In 2018, RISC-V had just started gaining popularity, raising the question: **Could it handle the required computation for navigation schemes?**

## My Role
- Tasked to implement an on-board computer (OBC) based on **RISC-V** to replace PIC32 in the Inertial Navigation System.  
- Implemented the solution on **Arty FPGA** using Freedom RISC-V implementation.  
- Integrated and tested navigation-related functionalities.  

## Outcome
- Successfully demonstrated the **feasibility of RISC-V** for navigation schemes.  
- Proved that RISC-V could serve as a long-term, flexible alternative to proprietary processors.
